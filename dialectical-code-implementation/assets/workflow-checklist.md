# 7步流程检查清单

## 概览
本文档提供了7步标准化流程的完整检查清单，用于在执行过程中验证每步的输出是否完整、准确、符合要求。

## 第一步：接收任务与初始调研

### 任务理解检查
- [ ] 是否明确了项目的业务背景？
- [ ] 是否明确了目标用户和使用场景？
- [ ] 是否明确了性能要求和约束条件？
- [ ] 是否明确了技术栈现状（编程语言、框架、依赖）？
- [ ] 是否明确了现有代码情况（代码结构、关键模块、技术债务）？
- [ ] 是否明确了需求细节（功能点、边界条件、优先级）？

### 信息收集检查
- [ ] 是否收集了所有必要的背景信息？
- [ ] 是否识别了信息缺口？
- [ ] 是否向用户确认了模糊需求？
- [ ] 是否记录了所有约束条件？

### 输出完整性检查
- [ ] 是否生成了任务理解报告？
- [ ] 报告是否包含项目背景？
- [ ] 报告是否包含技术栈现状？
- [ ] 报告是否包含需求概述？
- [ ] 报告是否包含初始问题清单？

---

## 第二步：初步构思与可行性分析

### 技术可行性检查
- [ ] 是否评估了现有技术栈能否满足需求？
- [ ] 是否识别了技术难点？
- [ ] 是否识别了潜在技术风险？
- [ ] 是否判断了是否需要引入新技术？

### 时间可行性检查
- [ ] 是否估算了各方案的实现时间？
- [ ] 是否评估了对现有功能的影响范围？
- [ ] 是否识别了关键路径和里程碑？

### 资源可行性检查
- [ ] 是否评估了人力需求？
- [ ] 是否评估了算力需求？
- [ ] 是否评估了存储需求？
- [ ] 是否识别了资源瓶颈？

### 方案构思检查
- [ ] 是否设计了至少2-3种实现方案？
- [ ] 是否对比了各方案的优劣势？
- [ ] 是否标注了方案的激进程度？

### 输出完整性检查
- [ ] 是否生成了可行性分析报告？
- [ ] 报告是否包含多方案对比？
- [ ] 报告是否包含时间估算？
- [ ] 报告是否包含资源需求？
- [ ] 报告是否包含初步风险清单？

### 决策点检查
- [ ] 是否判断了技术可行性？
- [ ] 是否判断了时间可行性？
- [ ] 是否判断了资源可行性？
- [ ] 对于不可行的情况，是否提出了替代方案？

---

## 第三步：评估"度"与风险预判

### 影响范围评估检查
- [ ] 是否评估了对现有代码架构的影响范围？
- [ ] 是否评估了对业务流程的影响程度？
- [ ] 是否评估了对性能的影响程度？
- [ ] 是否评估了对团队的认知负荷？

### 风险识别检查
- [ ] 是否识别了技术风险？
- [ ] 是否识别了时间风险？
- [ ] 是否识别了资源风险？
- [ ] 是否识别了业务风险？

### 质变预警判断检查
- [ ] 是否对照了质变预警触发标准？
- [ ] 是否评估了影响范围是否超过阈值？
- [ ] 是否评估了技术复杂度是否超出团队经验？
- [ ] 是否评估了潜在风险等级？

### 输出完整性检查
- [ ] 是否生成了度评估报告？
- [ ] 报告是否包含影响程度评级？
- [ ] 报告是否包含风险清单？
- [ ] 报告是否包含质变预警判断？

### 决策点检查
- [ ] 是否明确判断了是否触发质变预警？
- [ ] 对于触发的场景，是否准备进入第四步？
- [ ] 对于未触发的场景，是否准备进入第五步？
- [ ] 是否判断了风险是否可控？

---

## 第四步：深度辩证分析（仅质变预警触发时执行）

### 问题现象识别检查
- [ ] 是否描述了当前面临的具体问题？
- [ ] 是否描述了问题的表现形式？
- [ ] 是否描述了问题的触发条件？
- [ ] 是否描述了问题的影响范围和严重程度？

### 根本原因挖掘检查
- [ ] 是否进行了多维度分析（技术、流程、组织、业务）？
- [ ] 是否识别了表层原因？
- [ ] 是否识别了深层原因？
- [ ] 是否区分了直接原因和间接原因？
- [ ] 是否使用了5Why等分析方法？

### 主要矛盾识别检查
- [ ] 是否列举了所有矛盾？
- [ ] 是否对比了各矛盾的影响力权重？
- [ ] 是否分析了矛盾之间的制约关系？
- [ ] 是否识别了主要矛盾？

### 矛盾转化路径探索检查
- [ ] 是否分析了矛盾的转化条件？
- [ ] 是否寻找了矛盾转化的关键节点？
- [ ] 是否评估了转化路径的可行性？
- [ ] 是否设计了可行的转化路径？

### 输出完整性检查
- [ ] 是否生成了深度辩证分析报告？
- [ ] 报告是否包含问题现象描述？
- [ ] 报告是否包含根本原因分析？
- [ ] 报告是否包含主要矛盾识别？
- [ ] 报告是否包含转化路径建议？

### 决策点检查
- [ ] 是否确认主要矛盾被准确识别？
- [ ] 是否验证了矛盾分析的正确性？
- [ ] 是否找到了可行的矛盾转化路径？

---

## 第五步：探索扬弃路径并决策

### 方案对比检查
- [ ] 是否列出了所有可行方案？
- [ ] 是否构建了对比维度？
- [ ] 是否评估了各方案的优劣势？
- [ ] 是否进行了综合评分？

### 扬弃分析检查
- [ ] 是否明确了每个方案的"扬"（保留）内容？
- [ ] 是否明确了每个方案的"弃"（抛弃）内容？
- [ ] 是否明确了每个方案的"创"（创新）内容？
- [ ] 是否明确了扬弃的理由？

### 决策依据检查
- [ ] 是否明确了决策依据？
- [ ] 是否对比了各方案的综合得分？
- [ ] 是否考虑了关键约束条件？
- [ ] 是否考虑了风险可控性？
- [ ] 是否考虑了团队能力匹配？
- [ ] 是否考虑了长期价值？

### 实施策略检查
- [ ] 是否制定了实施计划？
- [ ] 是否明确了实施阶段？
- [ ] 是否明确了实施顺序？
- [ ] 是否设置了检查点？
- [ ] 是否制定了风险防范措施？

### 输出完整性检查
- [ ] 是否生成了扬弃决策报告？
- [ ] 报告是否包含方案对比表？
- [ ] 报告是否包含最终方案？
- [ ] 报告是否包含决策依据？
- [ ] 报告是否包含实施策略？

### 决策点检查
- [ ] 是否达成了决策共识？
- [ ] 是否确认方案可落地？
- [ ] 是否保留了决策过程的文档？

---

## 第六步：代码实现与单元测试

### 实现计划检查
- [ ] 是否细化了任务拆解？
- [ ] 是否明确了实现顺序？
- [ ] 是否设置了检查点？
- [ ] 是否明确了代码审查流程？

### 代码实现检查
- [ ] 是否提供了关键模块的实现思路？
- [ ] 是否定义了接口规范？
- [ ] 是否定义了数据结构？
- [ ] 是否标注了技术难点？
- [ ] 是否强调了代码质量要求？

### 单元测试检查
- [ ] 是否定义了测试用例？
- [ ] 是否覆盖了正常场景？
- [ ] 是否覆盖了边界场景？
- [ ] 是否覆盖了异常场景？
- [ ] 是否设计了测试数据？
- [ ] 是否明确了验收标准？

### 风险防范检查
- [ ] 是否制定了回滚方案？
- [ ] 是否设置了监控指标？
- [ ] 是否预留了调优空间？

### 输出完整性检查
- [ ] 是否生成了代码实现方案？
- [ ] 方案是否包含实现计划？
- [ ] 方案是否包含代码框架？
- [ ] 方案是否包含测试方案？
- [ ] 方案是否包含风险防范措施？

### 决策点检查
- [ ] 是否需要调整实现方案？
- [ ] 测试是否覆盖了所有关键场景？

---

## 第七步：交付代码并提供问题说明与发展预判

### 交付内容检查
- [ ] 是否整理了代码清单？
- [ ] 是否提供了配置说明？
- [ ] 是否提供了部署文档？
- [ ] 是否更新了技术文档？

### 问题说明检查
- [ ] 是否列出了已解决的问题？
- [ ] 是否说明了遗留问题？
- [ ] 是否提出了后续优化方向？
- [ ] 是否制定了技术债务偿还计划？

### 发展预判检查
- [ ] 是否预测了方案的演进路径？
- [ ] 是否识别了可能的新矛盾？
- [ ] 是否提出了应对策略？
- [ ] 是否考虑了技术发展趋势？

### 经验总结检查
- [ ] 是否提炼了关键经验？
- [ ] 是否总结了方法论层面的启示？
- [ ] 是否形成了可复用的实践模板？

### 输出完整性检查
- [ ] 是否生成了交付报告？
- [ ] 报告是否包含代码清单？
- [ ] 报告是否包含部署说明？
- [ ] 报告是否包含问题说明？
- [ ] 报告是否包含发展预判？
- [ ] 报告是否包含经验总结？

### 决策点检查
- [ ] 是否确认交付完整？
- [ ] 是否验证了预判的合理性？

---

## 整体流程检查

### 流程顺序检查
- [ ] 是否严格遵循了7步流程顺序？
- [ ] 每步的输出是否作为下一步的输入？
- [ ] 是否形成了完整的逻辑闭环？

### 质变预警处理检查
- [ ] 第三步触发质变预警时，是否执行了第四步？
- [ ] 第四步完成后，是否进入了第五步？
- [ ] 第三步未触发质变预警时，是否跳过了第四步？

### 文档完整性检查
- [ ] 每步是否都生成了相应的报告？
- [ ] 所有报告是否完整、准确？
- [ ] 决策依据是否充分、清晰？
- [ ] 风险提示是否全面、可操作？

### 决策点处理检查
- [ ] 所有决策点是否都已处理？
- [ ] 决策依据是否充分？
- [ ] 决策过程是否完整记录？

---

## 场景适配检查

### 简单场景（不触发质变预警）
- [ ] 跳过了第四步？
- [ ] 第五步决策是否基于第二、三步的输出？

### 复杂场景（触发质变预警）
- [ ] 执行了第四步？
- [ ] 第五步决策是否基于第二、三、四步的输出？

### 高风险场景（红色预警）
- [ ] 是否进行了外部专家评审？
- [ ] 是否进行了小规模试点？
- [ ] 是否制定了详细回滚方案？
- [ ] 是否设置了全方位监控？

---

## 最佳实践检查

### 风险管理
- [ ] 是否在每步都考虑了风险？
- [ ] 是否制定了风险应对措施？
- [ ] 是否设置了监控和告警？

### 质量保证
- [ ] 是否设置了检查点和验收标准？
- [ ] 是否进行了代码审查？
- [ ] 是否进行了充分的测试？

### 沟通协作
- [ ] 是否向利益相关者通报了进度？
- [ ] 是否及时沟通了风险？
- [ ] 是否保持了团队共识？

### 持续改进
- [ ] 是否记录了经验教训？
- [ ] 是否更新了知识库？
- [ ] 是否总结了最佳实践？
